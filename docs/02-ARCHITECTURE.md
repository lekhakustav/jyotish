# 02 — Architecture

## Stack
- **SwiftUI**, iOS 17+, single target `Jyotish`, generated by **XcodeGen** (`project.yml`).
- No third-party iOS packages. Astrology is computed on device; Supabase sync is optional
  and uses Foundation `URLSession` against Auth/PostgREST. Pandit-ji chat can call the
  local OpenAI backend, with on-device `PanditBrain` as fallback.
- Build: `xcodegen generate && xcodebuild -scheme Jyotish -destination 'platform=iOS Simulator,name=iPhone 17 Pro' build`

## State
- `AppState: ObservableObject` — single source of truth injected as `@EnvironmentObject`.
  Holds: `session` (logged-out / loggedIn(Profile)), `family: [FamilyMember]`,
  `events: [PatroEvent]`, `chat: [ChatMessage]`, `language: Language`, `theme: ThemeChoice`.
- Every mutation goes through `AppState` methods which persist via `DataStore`.

## Supabase sync
Every I/O path goes through protocols so the app can run local-only or sync with Supabase:

```swift
protocol AuthService  { func signIn(...) async throws -> UserAccount; func signOut() ... }
protocol DataStore    { func loadProfile() / saveProfile / loadFamily / saveFamily
                        loadEvents / saveEvents / loadChat / saveChat }
```
- `DummyAuthService` — instant fake account when Supabase is not configured.
- `LocalDataStore` — JSON files in `Documents/` (atomic writes, Codable), retained as
  offline cache even when Supabase is configured.
- `SupabaseAuthService` — anonymous Supabase Auth for the current "Continue" flow.
- `SupabaseDataStore` — persists the full `Household` aggregate to the user-owned
  `households.payload` JSONB row through PostgREST and RLS.

See `docs/07-SUPABASE.md` for the SQL schema, RLS policies, and key handling.

## Pandit agent backend
`AppState.sendChat(_:)` appends the user's message immediately, builds an
`AgentChatRequest` from the current household, and posts it through
`HTTPAgentService` to `JYOTISH_AGENT_BASE_URL` (default `http://127.0.0.1:8788`).
The backend is `server/jyotish-agent.mjs`, which reads the ignored `.env.local`
`OPENAI_API_KEY` and calls OpenAI server-side. If the server is unavailable or returns an
invalid reply, `AppState` appends the deterministic local `PanditBrain` answer instead.

## Profile gating
Kundali-dependent features (rashifal personalization, kundali screen, chat questions that
need a chart) check `profile.hasBirthData`. If missing → `ProfileGateView` prompt:
"Please complete the birth profile first" with a button straight to ProfileSetup.

## Persistence formats
Single `household.json` per account locally and one Supabase `households` row remotely:
`{ account, family, events, chat, language, theme }`, versioned (`schemaVersion: 1`)
for future migration.
